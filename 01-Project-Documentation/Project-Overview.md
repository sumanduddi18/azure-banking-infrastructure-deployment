# Project Overview

## Project Name

Azure Banking Infrastructure Deployment

---

## Objective

The objective of this project was to design, deploy, secure, monitor, and govern a production-style Azure infrastructure that simulates a banking environment using Microsoft Azure services and enterprise best practices.

The implementation focuses on practical Azure Administration and Azure Security concepts commonly used in enterprise cloud environments.

---

## Project Scope

The project includes the deployment and configuration of:

- Azure Resource Group
- Virtual Network (VNet)
- Web and Database Subnets
- Network Security Groups (NSGs)
- Windows Server Virtual Machine
- Public IP Address
- Azure Key Vault
- Managed Identity
- Azure Monitor
- Log Analytics Workspace
- VM Insights
- Metric Alerts
- Action Groups
- Recovery Services Vault
- Azure Backup
- Azure Policy
- Resource Tags
- Resource Locks
- Cost Optimization

---

## Business Scenario

ABC Bank is migrating one of its internal web applications from an on-premises data center to Microsoft Azure.

The organization requires a secure, scalable, and well-governed cloud infrastructure that follows enterprise operational standards.

The cloud platform must provide:

- Secure network isolation
- Identity-based access control
- Centralized monitoring
- Backup and recovery capabilities
- Resource governance
- Cost visibility
- Operational reliability

## Project Outcome

The project successfully demonstrates an end-to-end Azure Administration implementation covering infrastructure deployment, security, monitoring, governance, backup, and operational best practices.

The final environment closely resembles a real production Azure deployment and provides hands-on experience with Azure services commonly used by Azure Administrators and Azure Security Engineers.
